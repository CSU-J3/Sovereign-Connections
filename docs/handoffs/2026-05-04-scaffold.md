## Decisions worth flagging

1. **Component filenames are PascalCase**, not kebab-case. The original handoff said kebab-case but cbt itself uses PascalCase (`HeaderBar.tsx`, `BillRow.tsx`); followed cbt as the canonical reference.
2. **ASCII hyphens in JSON, en-dashes at render time.** Period strings stored as `2021-PRES`; `formatPeriod` converts to `2021–PRES` for display. Cross-system safety win.
3. **URL-based filter and expand state**, no `useState`. Filter clicks are server-rendered `Link` components that swap query parameters; `RecordRow` reads `?expanded=ID` from `searchParams`. Mirrors cbt.
4. **No `/record/[id]/walk` page.** The walk-the-definition button copies the prompt and opens `claude.ai/new` in a new tab. Simpler than a dedicated route and adds no surface to maintain.

## Deploy lessons (worth keeping for next time)

1. **Vercel CLI deploy from `web/` uploaded only `web/` contents.** The original parent-directory data import (`../../data/*.json` with `outputFileTracingRoot` pointing at the repo root) failed at build time because parent files were not in the upload context. Fix: switched to GitHub-integration deploy, then later moved `data/*.json` into `web/data/` and dropped `outputFileTracingRoot` entirely (Fix A from the original handoff). Preferred long-term: collapse the deploy boundary to match the project boundary.
2. **Initial GitHub import did not apply Root Directory = `web`.** Build log showed `Build Completed in /vercel/output [56ms]` with no `npm install` step and no Next.js compile, meaning `vercel build` ran against the repo root, found no Next.js project, and shipped an empty output. Every URL then returned a Vercel-level 404 (not a Next.js 404). Fix: Settings → General → Root Directory → `web` → redeploy.
3. **Setting Root Directory reset Framework Preset to non-Next.js.** Subsequent build succeeded with all 12 routes compiled, but post-build looked for `public/` instead of `.next/` and failed. Fix: Settings → Build & Development Settings → Framework Preset → Next.js → redeploy.
